# 녹음 파일 텍스트 변환 작업 명세

이 문서는 `녹음 파일 텍스트 변환` 기능을 다시 수정하거나 이어서 개선할 때 확인해야 할 작업 명세다. 이 기능은 코워크스페이스의 브라우저 비저장 원칙을 유지하면서, 휴대폰 녹음 파일을 검토용 텍스트 초안으로 바꾸는 베타 도구다.

## 기능 목표

- 사용자가 선택한 m4a, mp3, wav, aac, webm, ogg 녹음 파일을 서버 업로드 없이 브라우저 안에서 텍스트로 변환한다.
- 결과는 최종 회의록이나 법적 기록이 아니라 사람이 다시 확인해야 하는 초안으로 취급한다.
- 저장하지 않는 가벼운 도구라는 장점을 분명히 보여 주되, 서버형 대형 STT보다 품질이 낮을 수 있다는 한계도 UI에 명확히 표시한다.
- 한국어 ARS, 안내 문구, 휴대폰 녹음 파일에서 `whisper-tiny` 품질이 부족한 경우가 확인되어 `정확도 우선` 모델만 사용자에게 제공한다.

## 현재 구현 기준

- 렌더러: `app.js`의 `renderAudioFileTranscription`
- 도구 정의: `app.js`의 `TOOL_DEFS` 중 `audio-file-transcription`
- 모델 프로필:
  - `quality`: `onnx-community/whisper-base`, 기본값, `정확도 우선`
  - `whisper-tiny` 기반 빠른 변환 fallback은 품질 기준 미달로 UI와 프로필에서 제외한다.
- 실행 방식:
  - UI에는 모델 선택지를 노출하지 않는다. 품질 기준을 통과한 `정확도 우선` 경로만 제공한다.
  - `정확도 우선`은 더 큰 모델을 쓰므로 첫 실행 다운로드와 처리 시간이 길 수 있다.
  - 기본 품질 경로는 호환성이 검증된 WASM mixed precision을 사용한다.
  - 모델 다운로드와 음성 인식 추론은 Web Worker에서 실행해 메인 페이지가 응답 없음 상태로 빠지지 않게 한다.
  - 메인 스레드에서 녹음 파일을 16 kHz 원시 파형으로 디코딩한 뒤 Worker로 전달한다. Worker 환경에는 AudioContext가 없을 수 있으므로 Blob URL을 Worker 안에서 다시 읽는 방식은 사용하지 않는다.
  - WebGPU 고품질, CPU 고품질 같은 불안정한 명시 옵션은 UI에 노출하지 않는다.
- 입력 제한:
  - 최대 5분 권장
  - 최대 80MB
- 출력 정리:
  - `문장 끝 기준 줄바꿈` 옵션을 기본 체크한다.
  - `.`, `?`, `!`, `。`, `！`, `？` 기준 줄바꿈을 지원한다.
  - 연속으로 반복되는 짧은 문장은 초안 렌더링 전에 제거한다.
- 로컬 오디오 전처리:
  - 선택한 녹음 파일을 브라우저에서 디코딩해 mono PCM으로 변환한다.
  - STT 입력 전 16 kHz 원시 파형으로 준비한다.
  - 대화 중간의 낮은 볼륨 발화와 쉬는 구간은 유지하고, 앞뒤의 확실한 빈 구간만 보수적으로 정리한다.
  - 작은 목소리는 RMS 기준으로 자동 증폭하되, 과한 피크는 모델 전달 전에 제한한다.
  - 디코딩 실패 시 Worker에서 원본 Blob URL을 다시 읽지 않고 사용자에게 재시도 가능한 오류를 안내한다.

## UI 문구 기준

- 기능명 옆에는 반드시 빨간색 `(베타)`를 붙여 쓴다.
- `(베타)` 앞에 공백을 넣지 않는다. 예: `녹음 파일 텍스트 변환(베타)`
- `(베타)`는 줄바꿈을 유도하지 않아야 하며, CSS에서 빨간색을 유지한다.
- 비저장 방식 안내는 사용자가 파일을 선택하기 전에 보여야 한다.
- 품질 한계 안내는 숨기지 않는다. 핵심 문구는 다음 의미를 유지한다.
  - 파일은 코워크스페이스 서버로 업로드하지 않는다.
  - 비저장 브라우저 처리라 전문 서버형 음성 인식보다 품질이 좋지 않을 수 있다.
  - 중요한 내용은 사람이 다시 확인해야 한다.

## 품질 관련 판단 기준

- 브라우저 안에서 처리하는 한 서버형 STT와 같은 품질을 보장할 수 없다.
- 품질이 낮다는 피드백이 있으면 우선 확인할 항목은 다음 순서다.
  - 실제 운영 페이지가 최신 `app.js` 캐시 버전을 쓰는지
  - 기본 모델이 `quality`인지
  - `onnx-community/whisper-base` 세션이 실제로 준비되는지
  - `whisper-tiny` 또는 빠른 변환 선택지가 사용자 UI에 남아 있지 않은지
  - WebGPU 사용 가능 여부와 CPU fallback 여부
  - 음성 파일의 잡음, 발화 거리, 통화 압축, 다중 화자 여부
- 더 큰 품질 개선이 필요하면 브라우저-only 원칙을 유지할지, 선택형 서버 STT를 별도 기능으로 둘지 먼저 사용자에게 확인해야 한다.
- `onnx-community/whisper-base`는 전처리와 반복 제거 옵션을 보강한 뒤 실제 휴대폰 녹음에서 `whisper-tiny`보다 문맥을 더 잘 살리는 것이 확인되어 기본 `정확도 우선` 모델로 복귀했다.

## 보안과 개인정보

- 녹음 파일 원본, 파일명, 변환 결과를 분석 이벤트나 외부 API로 보내지 않는다.
- 모델 파일은 CDN과 Hugging Face에서 내려받을 수 있지만, 사용자 녹음 파일은 코워크스페이스 서버에 업로드하지 않는다.
- 오디오 전처리 결과도 브라우저 Object URL로만 다루고 서버로 보내지 않는다.
- 전처리된 원시 파형 버퍼는 Web Worker로 전달하고 서버로 보내지 않는다.
- Object URL은 미리듣기 등 필요한 경우에만 만들고 사용 후 revoke한다.
- 오류 문구에는 사용자가 확인할 수 있는 네트워크, 브라우저, 실행 방식 힌트를 포함한다.

## 변경 시 필수 작업

- `app.js` 또는 `styles.css`가 바뀌면 모든 HTML, `site.webmanifest`, `PROJECT_SPEC.md`, `ROADMAP.md`의 정적 리소스 캐시 버전을 올린다.
- UI 문구가 바뀌면 `PROJECT_SPEC.md`, `README.md`, `ROADMAP.md` 중 관련 설명도 함께 맞춘다.
- 모델 프로필, fallback 순서, 오류 안내가 바뀌면 `scripts/smoke-test.js`에 회귀 검사를 추가하거나 갱신한다.
- 작업 후 `npm.cmd run check`를 통과시킨다.
- 배포가 필요한 변경이면 커밋, `origin/main` 푸시, GitHub Actions 성공, 운영 URL 반영을 확인한다.

## 최근 작업 상태

- `whisper-base` 품질 프로필은 전처리와 반복 제거를 보강한 뒤 실제 녹음 재검증에서 `whisper-tiny`보다 문맥을 더 잘 살려 기본값으로 복귀했다.
- `whisper-tiny`는 ARS 음성에서도 품질 기준을 넘지 못해 빠른 변환 fallback에서 제거했다.
- 녹음 파일 변환 기능의 베타 표시와 비저장 품질 한계 안내를 UI에 추가했다.
- 연속 반복 문장 제거 후처리를 추가했다.
- 1순위 개선안으로 16 kHz WAV 준비, 앞뒤 빈 구간 정리, 작은 목소리 보정 전처리를 추가했다.
- 실제 휴대폰 녹음에서 작은 발화가 누락되는 문제가 확인되어 내부 무음 압축은 기본 전처리에서 제거하고 대화 흐름 보존을 우선한다.
- 한국어 ARS/안내 음성에서 `whisper-tiny` 품질이 현저히 부족해 `whisper-base`를 정확도 우선 모델로 올리고, `whisper-tiny` 선택지는 제거했다.
- `whisper-base` 추론 중 브라우저 응답 없음 경고가 확인되어 STT 모델 로딩과 변환을 Web Worker로 분리했다.
- Worker는 Blob URL 대신 메인 스레드에서 디코딩한 원시 파형 데이터를 받아 처리한다. 실제 휴대폰 녹음 파일 기준으로 변환 중 브라우저 응답 샘플을 지속 수집했고 완료 결과를 확인했다.
