    CSV 엑셀 변환기 시나리오

    자세히 보기

    CSV 엑셀 변환기는 명단, 성적표, 주문 내역, 시스템 추출 파일처럼 CSV와 XLSX를 오가며 제출용 자료를 만들어야 하는 상황을 줄이기 위한 도구입니다. CSV를 그냥 엑셀로 열면 한글 인코딩이 맞지 않아 글자가 깨지거나, 전화번호와 우편번호 앞자리 0이 사라지거나, 날짜처럼 보이는 값이 자동 변환되는 일이 자주 생깁니다. 이 도구는 CSV와 TSV 파일을 읽어 XLSX로 만들고, XLSX 파일의 시트를 CSV로 다시 내보내는 반복 작업을 브라우저 안에서 처리합니다.

    CSV 읽기 단계에서는 UTF-8과 CP949/EUC-KR을 선택하거나 자동 감지할 수 있습니다. 쉼표, 탭, 세미콜론 구분자도 자동 감지하거나 직접 지정할 수 있어 기관 시스템이나 업무 프로그램에서 내려받은 파일을 다시 열어 확인하기 좋습니다. 앞자리 0, 긴 숫자, 날짜처럼 보이는 텍스트를 그대로 보존하는 옵션을 기본으로 켜 두어 원본 값이 마음대로 바뀌는 문제를 줄였습니다.

    여러 파일을 한 번에 선택하면 각 파일별 결과를 만들고, 개별 다운로드하거나 ZIP으로 묶어 저장할 수 있습니다. XLSX를 CSV로 내보낼 때는 첫 시트만 저장하거나 모든 시트를 각각 CSV로 저장할 수 있습니다. CSV는 색상, 차트, 병합셀 같은 엑셀 서식을 담지 못하므로 결과는 셀 값 중심으로 확인하는 것이 좋습니다.

    선택한 파일 내용과 변환 결과는 코워크스페이스 자체 서버로 업로드하지 않습니다. 스프레드시트 변환 라이브러리 파일만 필요할 때 CDN에서 내려받고, 사용자가 넣은 파일명, 셀 내용, 결과 데이터는 분석 이벤트나 광고 이벤트로 보내지 않습니다. 민감한 명단이나 학습자료를 다룰 때도 결과를 다운로드하기 전 미리보기에서 앞부분을 확인하고, 최종 제출 전에는 실제 업무 시스템에서 한 번 더 여는 흐름을 권장합니다.

    이 도구에서 특히 신경 쓴 부분

    • CSV와 TSV를 XLSX로, XLSX를 CSV로 변환하는 흐름을 한 화면에 묶었습니다.
    • 한글 CSV 인코딩과 앞자리 0 보존처럼 한국 실무에서 자주 생기는 문제를 기본 옵션으로 다룹니다.
    • 여러 파일을 처리하고 결과를 ZIP으로 저장할 수 있어 반복 변환 시간을 줄입니다.
