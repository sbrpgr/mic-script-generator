# 코워크스페이스 에이전트 지침

이 저장소에서 작업하는 에이전트는 빠른 수정만 보지 말고, 정적 배포, 개인정보 보호, 광고/SEO 영향, 캐시 갱신, 검증까지 함께 본다.

## 먼저 읽을 문서

- `AGENT_OPERATING_SPEC.md`: 전체 에이전트 운영 기준
- `CHANGE_PLAYBOOK.md`: 기능, 광고, 분석, SEO 변경 체크리스트
- `PROJECT_SPEC.md`: 현재 제품 범위와 구현 상태
- `AUDIO_TRANSCRIPTION_WORK_SPEC.md`: 녹음 파일 텍스트 변환 베타 기능의 상세 작업 명세

## 기본 작업 규칙

- 시작할 때 `git status --short`로 작업 트리를 확인한다.
- 미추적 파일이나 사용자 작업 파일은 요청 없이 건드리지 않는다.
- `app.js` 또는 `styles.css`를 수정하면 정적 리소스 캐시 버전을 함께 올린다.
- 사용자 입력, 파일명, 변환 결과, 녹음 내용은 분석 이벤트나 외부 API로 보내지 않는다.
- 광고 영역은 입력, 업로드, 변환, 결과 UI 안에 넣지 않는다.
- 기능 변경 후에는 `npm.cmd run check`를 실행한다.

## 녹음 파일 텍스트 변환 주의사항

- 이 기능은 베타이며 브라우저 비저장 방식이다.
- 제목 표기는 `녹음 파일 텍스트 변환(베타)`처럼 `(베타)`를 붙여 쓴다.
- `(베타)`는 반드시 빨간색이어야 하며, 앞에 공백을 넣어 줄바꿈 지점을 만들지 않는다.
- 기본 모델은 `onnx-community/whisper-tiny` 안정 프로필이다.
- `onnx-community/whisper-base`는 실제 녹음에서 반복 환각이 확인되어 기본값이나 노출 옵션으로 다시 올리지 않는다.
- 품질 개선은 먼저 브라우저 안 전처리, 대화 흐름 보존, 앞뒤 빈 구간 정리, 작은 목소리 볼륨 보정, 반복 문장 정리부터 검토한다.
- 품질 한계 안내와 사람이 다시 확인해야 한다는 안내를 숨기지 않는다.

## 완료 보고 기준

완료 보고에는 변경 내용, 검증 결과, 배포 여부, 남은 리스크를 짧게 남긴다. 배포했다면 커밋 해시와 운영 URL 확인 결과를 함께 적는다.
